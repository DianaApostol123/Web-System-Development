1. Abstract

This Scholarly HTML document describes the Web system internals of the Museum Web Guide (MUG) Application.


2. Introduction

The application's purpose is to properly assist a museum visitor. It can be downloaded via an app store (Magazin Play) by using the QR code printed on the museum ticket. The application will take advantage of the position of the user in the museum, using RFID, in order to determine where the person is and provide her/him useful audio and visual information based on existing data about Romanian museums. The content is available in 2 different languages: romanian and english. If a visitor wants to find out more, (s)he would walk to certain exhibits in order to have access to details of interest. At the end of the tour, the application will offer an overview and the photo album (s)he took along the way presented as time line. This work – plus the user comments/reviews regarding the tour – could be also shared on several significant social networks. Also the application includes for the review page, the possibility of interacting using different languages.


3. Structure

In the following section the Web system internals of the MUG Application.

3.1. QR Code

The QR code will be done using The METAQRCODE API, which provides developers a way to create their own XML metadata and put it inside a QRCode. This API is Rest based, and can be consumed using JSON or XML. We are using JSON. We used this because it is a Single purpose API. It is easier to develop and because of the popularity of the tool, the support is more easy to find.

3.2. RFID

In order for the app to know the user positioning in the museum, the RFID technology will be used. There will be for each area of the museum an RFID transmitor. The app will read the data from the RFID and will show to the user the informations from that section. This will be done using Technology Solutions (UK) Ltd. which sells the 1128 Bluetooth UHF RFID reader that comes with a software development kit so you can write your own apps.

Is designed to read and write to tags compliant with the EPC Gen 2 and ISO 18000-6C standard, and incorporates TSL's unique ASCII protocol, which provides a developer with a set of commands that carry out multiple actions locally within the Bluetooth reader.

3.3. Database

The database used is PostgresSql. The main reason is that it is a very good tool when it comes to integration with other tools. It handles data integrity and complex operations with ease. Another reasons are: it's open source, supports image, video, audio storage and also supports graphical data, it has high availability and is very easy to learn.

3.4. Instant Translate bot

For the Review page we will use a bot that will translate all the text. The Instant Bot can translate anything between 104 languages. This will increase the users interaction. If the user doesn't speak the language of another user, there is a button Translate which will display a pop up in order for the user to choose the desired language. After that the bot will change the text to that language. This will happen only for this user, the text will remain the same for the others.

3.5. Web Server

The server that we will use is Apache Web Server because the source code of Apache is available for free to anyone and no license is required. Also it can be installed easily and it is highly reliable and performs better.

3.6. Node.js

Node.js is a JavaScript runtime environment which lets you easily build server-side applications. It's unifying web-application development around a single programming language, rather than different languages for server- and client-side scripts. The advantages are: easy to learn, is used as a Single Programming Language, the support of large and active community, it's known for offering high performance

3.7. External Data

The informations will be taken from the site data about Romanian museums and Muzee si Colectii din Romania. The data will be stored in our database, a PostgreSql Database. We will use API to take the data from there into our application.
